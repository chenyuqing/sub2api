(Claude Code / SDK) participant MW as 🔐 中间件链 participant GW as 🚪 GatewayHandler participant CC as 🔒 ConcurrencyHelper participant SCH as 🎯 Scheduler participant FWD as 📡 Forwarder participant UP as 🌍 上游 AI API participant USG as 📊 Usage Recorder participant Redis as 🗄️ Redis C->>MW: POST /v1/messages
Header: x-api-key MW->>MW: RequestBodyLimit → ClientRequestID MW->>MW: OpsErrorLogger → EndpointNormalizer rect rgb(25,35,50) Note over MW,Redis: API Key 多级认证 MW->>MW: decodeAPIKey(x-api-key) MW->>Redis: L1 内存 LRU 缓存查询 alt 缓存命中 Redis-->>MW: apiKey + AuthSubject else 缓存未命中 MW->>Redis: L2 Redis 查询 Redis-->>MW: apiKey + AuthSubject end MW->>MW: 解析 user · group · subscription end MW->>MW: RequireGroupAssignment rect rgb(30,40,50) Note over GW,CC: 并发控制 GW->>CC: 检查等待队列 (最大排队数) CC->>CC: 获取用户并发槽位 CC->>CC: 计费资格检查 (余额/订阅/配额) end rect rgb(30,40,50) Note over GW,SCH: 粘性会话 + 账户调度 GW->>Redis: 查询粘性会话 (hash → account_id) GW->>SCH: 负载感知账户选择 SCH->>SCH: 模型可用性检查 SCH->>SCH: 账户槽位获取 SCH->>SCH: 用户消息队列 (OAuth 串行化) end rect rgb(35,30,45) Note over GW,UP: 请求转发 GW->>FWD: 按平台路由 (Anthropic/OpenAI/Gemini/Antigravity) FWD->>FWD: TLS 指纹模拟 (utls) FWD->>FWD: 代理路由 (可选 forward/socks5) FWD->>UP: SSE 流式请求透传 UP-->>FWD: SSE 流式响应 FWD->>FWD: 响应头安全过滤 end rect rgb(30,35,40) Note over GW,USG: 后处理 GW->>Redis: 保存粘性会话绑定 (account_id) GW->>Redis: RPM 计数器递增 GW->>USG: 异步用量记录提交 (Worker Pool) GW->>GW: Ops 指标采集 end GW-->>C: SSE Streaming Response

3. 认证与支付流程

sequenceDiagram actor U as 👤 用户 participant FE as 🖥️ Vue Frontend participant Auth as 🔐 AuthHandler participant Pay as 💰 PaymentHandler participant Prov as 🏦 支付渠道 participant WH as 📨 WebhookHandler participant DB as 💾 PostgreSQL rect rgb(25,35,50) Note over U,DB: 认证流程 U->>FE: 登录页面 FE->>Auth: POST /api/v1/auth/login Auth->>DB: 验证凭据 alt 2FA 已启用 Auth-->>FE: { requires_2fa: true } FE->>U: 输入 TOTP 验证码 FE->>Auth: POST /api/v1/auth/login/2fa end Auth->>DB: 签发 access_token + refresh_token Auth-->>FE: { tokens, user_info } FE->>FE: localStorage 存储 tokens FE->>FE: Axios 拦截器附加 Bearer token loop Token 自动刷新 Note over FE,Auth: 401 响应 → POST /auth/refresh → 新 token end end rect rgb(30,40,50) Note over U,DB: 支付流程 U->>FE: 选择订阅方案 FE->>Pay: GET /api/v1/payment/plans Pay-->>FE: 可用方案列表 FE->>Pay: GET /api/v1/payment/channels Pay-->>FE: 可用支付渠道 U->>FE: 选择渠道 → 确认下单 FE->>Pay: POST /api/v1/payment/orders Pay->>DB: 创建订单记录 Pay->>Prov: 按渠道路由 (支付宝/微信/Stripe/EasyPay) Prov-->>Pay: 支付 URL / QR Code Pay-->>FE: { order_id, payment_url } end rect rgb(35,30,45) Note over U,WH: 支付回调 FE->>U: 展示 QR Code / 跳转支付页 U->>Prov: 完成支付 Prov->>WH: POST webhook (签名验证) WH->>WH: 验证签名 ✓ WH->>DB: 更新订单状态 WH->>DB: 执行履约 (增加余额 / 激活订阅) WH-->>Prov: 200 OK end FE->>Pay: GET /api/v1/payment/orders/:id Pay-->>FE: { status: "completed" } FE->>U: 显示支付成功
